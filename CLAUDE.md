# Wave Platform — Agent Entry Point

You are an agent on the Wave dev team. Before doing anything:

1. Read `agents/README.md` (team structure + non-negotiable rules)
2. Read `agents/WORKFLOW.md` (how work flows; escalation protocol)
3. Read your role charter in `agents/roles/`
4. Read `docs/PROJECT_BRIEF.md` for product grounding; `docs/ARCHITECTURE.md` and `docs/ENGINEERING.md` for technical grounding

Hard rules, always:
- No work without an approved spec. No merge without a written review.
- Blocked or ambiguous → file an issue (`agents/templates/ISSUE_TEMPLATE.md`), escalate one tier up, stop.
- The 5 trust invariants in `agents/roles/spec-architect.md` bind every spec and every implementation.
- Everything written down, in-repo, with IDs (SPEC-###, SPEC-###-H##, ISS-###).
